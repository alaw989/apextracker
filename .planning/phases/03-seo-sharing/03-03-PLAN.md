<task type="auto">
  <name>Task 2: Add semantic HTML to PlayerView</name>
  <files>src/views/PlayerView.vue</files>
  <action>
    Update PlayerView.vue with semantic HTML elements.

    Required changes:
    1. Wrap content in <article> element (represents a self-contained composition)
    2. Use <header> for player info section
    3. Use <section> for stats and legends areas
    4. Add aria-labelledby to sections
    5. Use <h1> for player name (only h1 on this page)

    Structure:
    ```html
    <article>
      <header>
        <h1>{{ player.name }}</h1>
        <!-- avatar, rank info -->
      </header>

      <section aria-labelledby="stats-heading">
        <h2 id="stats-heading" class="visually-hidden">Player Statistics</h2>
        <!-- stats content -->
      </section>

      <section aria-labelledby="legends-heading">
        <h2 id="legends-heading" class="visually-hidden">Favorite Legends</h2>
        <!-- legends content -->
      </section>
    </article>
    ```

    Visually hidden class for accessibility:
    ```css
    .visually-hidden {
      position: absolute;
      width: 1px;
      height: 1px;
      padding: 0;
      margin: -1px;
      overflow: hidden;
      clip: rect(0, 0, 0, 0);
      border: 0;
    }
    ```

    DO NOT use itemsprop here - Schema.org markup is added in plan 03-06
    DO NOT forget to add visually-hidden class to styles
  </action>
  <verify>
    PlayerView.vue contains:
    - <article> wrapper element
    - <header> for player info
    - <h1> for player name
    - <section> elements with aria-labelledby
    - .visually-hidden class in styles
  </verify>
  <done>
